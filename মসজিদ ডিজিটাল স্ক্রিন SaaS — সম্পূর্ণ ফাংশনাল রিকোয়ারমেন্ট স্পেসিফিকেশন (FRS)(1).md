# **মসজিদ ডিজিটাল স্ক্রিন SaaS — সম্পূর্ণ ফাংশনাল রিকোয়ারমেন্ট স্পেসিফিকেশন (FRS)**

**প্রজেক্টের নাম:** Mosque Digital Display & Management SaaS  
**তারিখ:** ২১ এপ্রিল ২০২৬

**ভার্সন:** 1.0

---

## **১. সিস্টেম ওভারভিউ**

### **১.১ উদ্দেশ্য**

সারা বিশ্বের মসজিদগুলোকে একটি সম্পূর্ণ ডিজিটাল ইকোসিস্টেম দেওয়া — যেখানে টিভি স্ক্রিনে নামাযের সময়, ঘোষণা, ডোনেশন কার্যক্রম চলবে এবং মসজিদ প্রশাসন মোবাইল অ্যাপ থেকে সবকিছু নিয়ন্ত্রণ করবে। সাবস্ক্রিপশন ভিত্তিক SaaS মডেলে চলবে।

### **১.২ প্ল্যাটফর্ম ব্রেকডাউন**

| প্ল্যাটফর্ম | টেকনোলজি | উদ্দেশ্য |
| ----- | ----- | ----- |
| সুপার অ্যাডমিন প্যানেল (ওয়েব) | React.js | পুরো SaaS প্ল্যাটফর্ম নিয়ন্ত্রণ |
| মসজিদ অ্যাডমিন অ্যাপ (মোবাইল) | React Native | নির্দিষ্ট মসজিদ ম্যানেজমেন্ট |
| টিভি ডিসপ্লে সিস্টেম (ওয়েব) | React.js (Fullscreen) | স্মার্ট টিভি / ফায়ারস্টিক / Android TV-তে ব্রাউজার কিয়স্ক মোডে চলবে |
| ব্যাকএন্ড / রিয়েলটাইম | Firebase (Firestore \+ Auth \+ FCM) | ডাটা সিংক, অথেনটিকেশন, পুশ নোটিফিকেশন |
| পেমেন্ট | Stripe \+ PayPal | সাবস্ক্রিপশন বিলিং ও ডোনেশন প্রসেসিং |

## **২. ইউজার রোল ও পারমিশন**

* স্ক্রিন থিম ও লেআউট কাস্টমাইজ করা  
* ডোনেশন ক্যাম্পেইন তৈরি করা  
* একাধিক স্ক্রিন ম্যানেজ করা

### **২.৩ স্ক্রিন ডিভাইস (টিভি)**

* শুধুমাত্র Read-only অ্যাক্সেস  
* নিজ মসজিদের ডাটা রিয়েলটাইমে দেখাবে  
* কোনো এডিট/কন্ট্রোল করতে পারবে না

### **২.৪ পাবলিক ইউজার (ভবিষ্যত ফেজ)**

## **৩. সুপার অ্যাডমিন প্যানেল (ওয়েব ড্যাশবোর্ড)**

### **৩.১ ড্যাশবোর্ড হোম**

* মোট মসজিদ সংখ্যা (অ্যাক্টিভ / সাসপেন্ডেড / ট্রায়াল)  
* মোট অ্যাক্টিভ স্ক্রিন সংখ্যা  
* মাসিক রেভিনিউ (MRR) চার্ট  
* সাবস্ক্রিপশন ব্রেকডাউন (ফ্রি / বেসিক / প্রো / এন্টারপ্রাইজ)  
* সাম্প্রতিক রেজিস্ট্রেশন লিস্ট  
* এক্সপায়ারিং সাবস্ক্রিপশন সতর্কতা

### **৩.২ মসজিদ ম্যানেজমেন্ট**

**মসজিদ তৈরি করার সময় যা হবে:**

* মসজিদের নাম, ঠিকানা, দেশ, টাইমজোন ইনপুট  
* স্বয়ংক্রিয়ভাবে ইউনিক Mosque ID জেনারেট হবে (যেমন: MSQ-BD-001)  
* অ্যাডমিন অ্যাকাউন্ট অটো তৈরি হবে (ইমেইল \+ অটো-জেনারেটেড পাসওয়ার্ড)  
* সাবস্ক্রিপশন প্ল্যান অ্যাসাইন করা যাবে  
* অ্যাডমিনকে ইমেইলে লগইন ক্রেডেনশিয়াল পাঠানো হবে

**মসজিদ লিস্ট ভিউ:**

* সার্চ ও ফিল্টার (দেশ, প্ল্যান, স্ট্যাটাস অনুযায়ী)  
* প্রতিটি মসজিদের: নাম, দেশ, প্ল্যান, স্ট্যাটাস, স্ক্রিন সংখ্যা, লাস্ট অ্যাক্টিভ  
* অ্যাকশন: এডিট / সাসপেন্ড / ডিলিট / ইম্পারসোনেট (মসজিদ অ্যাডমিন হিসেবে লগইন)

* প্রতিটি স্ক্রিনের শেষ সিংক টাইম ও হার্টবিট

* জরুরি ঘোষণা সকল মসজিদে পুশ করা (ইমার্জেন্সি ব্রডকাস্ট)  
* কন্টেন্ট শিডিউলিং (তারিখ ও সময় অনুযায়ী)

## **৪. মসজিদ অ্যাডমিন মোবাইল অ্যাপ (React Native)**

### **৪.১ অথেনটিকেশন**

**লগইন মেকানিজম:**

* ইমেইল \+ পাসওয়ার্ড (Firebase Auth)  
* গুগল সাইন-ইন (ঐচ্ছিক)  
* ফোন নম্বর \+ OTP (ঐচ্ছিক, Firebase Phone Auth দিয়ে)  
* "পাসওয়ার্ড ভুলে গেছি" — Firebase Auth পাসওয়ার্ড রিসেট ইমেইল  
* প্রথম লগইনে পাসওয়ার্ড পরিবর্তন বাধ্যতামূলক  
* JWT Token ভিত্তিক সেশন

### **৪.২ নামাযের সময় ম্যানেজমেন্ট**

**অটোমেটিক (API ভিত্তিক):**

* **Aladhan API** থেকে নামাযের সময় অটো ফেচ (ফ্রি, কোনো API key লাগে না)  
* মসজিদের অবস্থান (latitude/longitude) অনুযায়ী সময় ক্যালকুলেট হবে  
* ক্যালকুলেশন মেথড সিলেকশন:  
  * কারাচি (বাংলাদেশ, পাকিস্তান, ভারত)  
  * MWL — মুসলিম ওয়ার্ল্ড লিগ (ইউরোপ)  
  * ISNA — ইসলামিক সোসাইটি অব নর্থ আমেরিকা  
  * উম্মুল কোরা (সৌদি আরব)  
  * মিশরীয় জেনারেল অথরিটি  
  * দিয়ানেত (তুরস্ক)  
  * আরও ১০+ মেথড  
* আসর জুরিস্টিক মেথড: শাফেয়ী/মালেকী/হাম্বলী অথবা হানাফী  
* হাই ল্যাটিচুড অ্যাডজাস্টমেন্ট (স্ক্যান্ডিনেভিয়া, উত্তর ইউরোপের জন্য)

**ম্যানুয়াল ওভাররাইড:**

* যেকোনো ওয়াক্তের সময় ম্যানুয়ালি পরিবর্তন করা যাবে  
* ম্যানুয়াল সময় সেট করলে সেই ওয়াক্তে API ওভাররাইড হবে

**ইকামাহ সেটিংস:**

* দুই ধরনের ইকামাহ সেটিং:  
  * **অফসেট মোড:** আযানের X মিনিট পরে ইকামাহ (যেমন: ফজর আযানের ২০ মিনিট পরে)  
  * **ফিক্সড টাইম মোড:** নির্দিষ্ট সময়ে ইকামাহ (যেমন: জোহর ইকামাহ সবসময় ১:৩০)  
* প্রতিটি ওয়াক্তের জন্য আলাদা আলাদা সেটিং  
* জুমুআর জন্য আলাদা সেটিং (খুতবা শুরু \+ নামায শুরু)  
* রমজানের জন্য বিশেষ সেটিং (তারাবীহ ইকামাহ)

**৫ ওয়াক্ত \+ অতিরিক্ত:**

* ফজর, সূর্যোদয় (ইশরাক), জোহর, আসর, মাগরিব, ইশা  
* জুমুআ (শুক্রবার বিশেষ)

## **৮. সিকিউরিটি ও ডাটা আইসোলেশন**

### **৮.১ অথেনটিকেশন**

* **Firebase Authentication** (Email/Password, Google, Phone)  
* JWT Token ভিত্তিক সেশন  
* Custom Claims দিয়ে role ম্যানেজ (superAdmin, mosqueAdmin, viewer)  
* টোকেন এক্সপায়ারি ও অটো রিফ্রেশ

### **৮.২ অথোরাইজেশন (Firestore Security Rules)**

// প্রতিটি মসজিদের ডাটা শুধুমাত্র সেই মসজিদের অ্যাডমিন পড়তে/লিখতে পারবে  
match /mosques/{mosqueId}/{document=\*\*} {  
  allow read: if request.auth.token.mosqueId \== mosqueId;  
  allow write: if request.auth.token.mosqueId \== mosqueId   
                && request.auth.token.role \== 'mosqueAdmin';  
}

// স্ক্রিন ডিভাইস শুধু পড়তে পারবে  
match /mosques/{mosqueId}/{document=\*\*} {  
  allow read: if request.auth.token.screenMosqueId \== mosqueId  
               && request.auth.token.role \== 'screen';  
}

// সুপার অ্যাডমিন সব পড়তে ও লিখতে পারবে  
match /{document=\*\*} {  
  allow read, write: if request.auth.token.role \== 'superAdmin';  
}

### **৮.৩ মাল্টি-টেন্যান্ট ডাটা আইসোলেশন**

* প্রতিটি মসজিদের ডাটা /mosques/{mosqueId}/ এর অধীনে সম্পূর্ণ আলাদা  
* Security Rules দিয়ে ডাটাবেজ লেভেলে আইসোলেশন নিশ্চিত  
* এমনকি অ্যাপ্লিকেশন কোডে বাগ থাকলেও এক মসজিদ অন্য মসজিদের ডাটা দেখতে পারবে না
